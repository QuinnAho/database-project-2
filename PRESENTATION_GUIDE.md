# Course Project Presentation Guide
## Home Cleaning Service Management System

---

## Project Overview
Database-driven website for managing home-cleaning services for contractor Anna Johnson.

**Total Presentation Time**: 12-15 minutes (6-7 minutes each presenter + Q&A)

---

## PRESENTER 1: Client-Side Journey

### Introduction (30 seconds)
- Introduce project: "Home cleaning service management system"
- "I'll demonstrate the client experience from registration to payment"

### Demo Checklist

#### 1. Client Registration (1 minute)
- [ ] Navigate to registration page
- [ ] Fill in all required fields:
  - [ ] First name
  - [ ] Last name
  - [ ] Address
  - [ ] Phone number
  - [ ] Email
  - [ ] Credit card information
- [ ] Submit registration
- [ ] **Point out**: Unique client ID is generated
- [ ] Confirm registration success

#### 2. Client Login (30 seconds)
- [ ] Login with email/password
- [ ] Show client portal dashboard

#### 3. Submit Service Request (1.5 minutes)
- [ ] Click "New Service Request" or similar
- [ ] Fill in all fields:
  - [ ] Service address
  - [ ] Type of cleaning (select: basic/deep cleaning/move-out)
  - [ ] Number of rooms
  - [ ] Preferred date/time
  - [ ] Proposed budget ($)
  - [ ] Optional notes (e.g., "pet-friendly products only", "bring own vacuum")
- [ ] Upload 2-3 photos of home
- [ ] Submit request
- [ ] **Show**: Request appears in "My Requests" with status "Pending"

#### 4. View Submitted Requests (30 seconds)
- [ ] Navigate to "My Requests" section
- [ ] Show request status
- [ ] Display request details

#### 5. Receive & Review Quote (1 minute)
- [ ] Refresh/navigate to see new quote from Anna
- [ ] Open quote details
- [ ] **Show all quote information**:
  - [ ] Adjusted price from Anna
  - [ ] Scheduled time window
  - [ ] Anna's note explaining pricing
- [ ] Review options: Accept or Counter

#### 6. Negotiate Quote - Counter Offer (1 minute)
- [ ] Click "Counter Offer" or "Negotiate"
- [ ] Submit counter note (e.g., "Price is too high, can we do $150?" or "Need different time slot")
- [ ] Submit counter
- [ ] Wait for Anna's revised quote
- [ ] **Show**: Negotiation history is visible

#### 7. Accept Quote (30 seconds)
- [ ] View revised quote from Anna
- [ ] Click "Accept Quote"
- [ ] **Show**: Order is automatically created
- [ ] Display success message

#### 8. View Order (30 seconds)
- [ ] Navigate to "My Orders"
- [ ] Open order details
- [ ] **Show**: Order is linked to accepted quote
- [ ] Display service agreement details

#### 9. Receive Bill (1 minute)
- [ ] Navigate to "My Bills" or refresh dashboard
- [ ] View bill for completed service
- [ ] **Show bill details**:
  - [ ] Itemized charges
  - [ ] Total amount
  - [ ] Due date
- [ ] Review options: Pay or Dispute

#### 10. Dispute Bill (1 minute)
- [ ] Click "Dispute Bill"
- [ ] Enter dispute note (e.g., "Extra charges not discussed beforehand" or "Service incomplete")
- [ ] Submit dispute
- [ ] Wait for revised bill from Anna
- [ ] View revised bill with Anna's adjustments and explanation

#### 11. Pay Bill (1 minute)
- [ ] Click "Pay Now" on revised bill
- [ ] Enter/confirm credit card information
- [ ] Submit payment
- [ ] **Show**: Payment confirmation
- [ ] Display receipt/paid status

### Wrap-up (30 seconds)
- Highlight key client features:
  - Easy service request submission with photo uploads
  - Transparent negotiation process
  - Flexible payment and dispute options
  - Complete visibility into service status
- Transition: "Now my partner will show Anna's admin portal..."

## PRESENTER 2: Admin Side (Anna's Portal)

### Introduction (30 seconds)
- "I'll demonstrate Anna's administrative interface for managing all client requests, orders, and billing"
- Login to admin portal

### Demo Checklist

#### 1. Admin Login & Dashboard (1 minute)
- [ ] Login to Anna's admin account
- [ ] **Show dashboard overview**:
  - [ ] Pending service requests
  - [ ] Active orders
  - [ ] Payment statuses
  - [ ] Multiple clients (show realistic data)

#### 2. View All Service Requests (30 seconds)
- [ ] Navigate to "Service Requests" section
- [ ] Display list of pending requests from multiple clients
- [ ] Show filtering/sorting options (if available)

#### 3. Review Service Request Details (1 minute)
- [ ] Open the request submitted by Presenter 1
- [ ] **Review all details**:
  - [ ] Client information
  - [ ] Service address
  - [ ] Cleaning type
  - [ ] Number of rooms
  - [ ] Preferred date/time
  - [ ] Client's proposed budget
  - [ ] Special notes
  - [ ] **View uploaded photos** (2-3 photos from client)

#### 4. Rejection Option (30 seconds - Optional)
- [ ] **Briefly show**: "Reject Request" button
- [ ] Explain: Can reject with explanatory note
- [ ] Example scenarios: "Outside service area" or "Fully booked"
- [ ] Don't actually reject - move to quote creation

#### 5. Create & Send Quote (1.5 minutes)
- [ ] Click "Create Quote" or "Respond"
- [ ] **Fill in quote details**:
  - [ ] Adjust price (different from client's proposal)
  - [ ] Set scheduled time window (date/time range)
  - [ ] Add note explaining pricing (e.g., "Price adjusted based on square footage and deep cleaning requirements")
- [ ] Submit quote to client
- [ ] **Show**: Quote appears as "Sent" status

#### 6. View Client Counter-Offer (1 minute)
- [ ] Receive notification of client's counter-offer
- [ ] Open negotiation thread
- [ ] **Show negotiation history**:
  - [ ] Original quote
  - [ ] Client's counter note
  - [ ] Timestamp for each interaction
- [ ] Read client's feedback

#### 7. Revise Quote (1 minute)
- [ ] Click "Revise Quote"
- [ ] Update price based on client feedback
- [ ] Adjust time window if needed
- [ ] Add explanation note (e.g., "Reduced price, can offer 10% discount for first-time client")
- [ ] Send revised quote
- [ ] **Show**: Revised quote sent to client

#### 8. View Accepted Quote (30 seconds)
- [ ] Receive notification: Client accepted quote
- [ ] **Show**: Order is automatically generated
- [ ] View order confirmation

#### 9. View All Orders (30 seconds)
- [ ] Navigate to "Orders" section
- [ ] Display list of orders:
  - [ ] Active orders
  - [ ] Completed orders
  - [ ] Order statuses

#### 10. Mark Service Complete (30 seconds)
- [ ] Open the relevant order
- [ ] Click "Mark as Completed"
- [ ] Change status to "Completed"
- [ ] Confirm service completion

#### 11. Generate Bill (1 minute)
- [ ] Click "Generate Bill" for completed order
- [ ] **Fill in bill details**:
  - [ ] Base service charge
  - [ ] Additional charges (if any)
  - [ ] Total amount
  - [ ] Due date
- [ ] Send bill to client
- [ ] **Show**: Bill is linked to order

#### 12. View Bill Dispute (1 minute)
- [ ] Receive client's bill dispute notification
- [ ] Open bill details
- [ ] Read client's dispute note
- [ ] **Show bill history/versions** (original bill visible)

#### 13. Revise Bill (1 minute)
- [ ] Click "Revise Bill"
- [ ] Adjust charges:
  - [ ] Remove disputed charge or apply discount
  - [ ] Update total
- [ ] Add explanation note (e.g., "Removed extra room charge, applied 15% discount for inconvenience")
- [ ] Send revised bill
- [ ] **Show**: New bill version created

#### 14. View Payment Received (30 seconds)
- [ ] Receive payment notification
- [ ] **Show payment status**: "Paid"
- [ ] View transaction details:
  - [ ] Payment date/time
  - [ ] Amount received
  - [ ] Payment method

#### 15. Show Database Evidence - CRITICAL (1.5 minutes)
- [ ] Navigate to database view or run query
- [ ] **Show all quote versions stored**:
  - [ ] Original quote
  - [ ] Revised quote(s)
  - [ ] Timestamps
  - [ ] All notes
- [ ] **Show all bill versions stored**:
  - [ ] Original bill
  - [ ] Revised bill(s)
  - [ ] Timestamps
  - [ ] All adjustments
- [ ] **Emphasize**: "This provides complete audit trail for any disputes or legal issues"

### Wrap-up & Technical Highlights (1 minute)
- **Key features demonstrated**:
  - Complete request-to-payment workflow
  - Bidirectional negotiation system
  - Database audit trail for accountability
  - Photo upload capability
  - Flexible billing and dispute resolution
- **Technical achievement**: "All quote and bill responses stored as evidence for disputes"
- **Business value**: "Provides transparency and protection for both client and contractor"

- [ ] **Presenter 1**: Have client account registered and ready

- [ ] Pre-load dummy data:
  - [ ] 2-3 other registered clients

2. **After P2 sends quote** -> P1 sees it in client portal
3. **After P1 counters quote** -> P2 sees counter-offer
4. **After P2 revises quote** -> P1 sees revised quote
5. **After P1 accepts** -> P2 sees order creation
6. **After P2 generates bill** -> P1 sees bill
7. **After P1 disputes** -> P2 sees dispute
8. **After P2 revises bill** -> P1 sees revised bill
9. **After P1 pays** -> P2 sees payment confirmation

## Presentation Tips

### For Both Presenters:
1. **Practice the handoffs** - Make transitions smooth
2. **Speak while clicking** - Explain what you're doing
3. **Go slowly** - Don't rush through forms
4. **Highlight key features** - Point out important functionality
5. **Use realistic data** - Make it look professional
6. **Test everything beforehand** - Multiple times!

### What to Emphasize:
- **User experience**: Easy to use for clients
- **Business value**: Efficient for Anna to manage
- **Data integrity**: Everything stored for accountability
- **Flexibility**: Negotiation allows fair pricing
- **Transparency**: Both parties see full history

### What to Avoid:
- Don't apologize for UI/styling (focus on functionality)
- Don't spend time on bugs/errors (have backup plan)
- Don't skip the database evidence section (most critical!)
- Don't go over time
